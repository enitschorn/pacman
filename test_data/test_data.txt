Here are some test scenarios that can be performed on the app.

Each test scenario is a list of commands that should be inputted in sequence, and an expected result.

Each scenario assumes that the app hasn't been started yet, and that no commands were issued yet.

---------------------
TEST 1: ignores commands before pacman is put into place

command list:
MOVE
LEFT
RIGHT
REPORT

result: output will be 'not in place', showing that the commands had no effect.

---------------------

TEST 2: place pacman on a given position
commands:
PLACE 1,2,NORTH
REPORT

expected result: '1,2,NORTH', pacman is in the correct position

---------------------

TEST 3: pacman is not placed in an invalid position
commands:
PLACE 9,9,NORTH
REPORT

expected result: 'not in place', invalid position was ignored

---------------------

TEST 4: pacman rotates right
commands:
PLACE 1,1,NORTH
RIGHT
REPORT
RIGHT
REPORT
RIGHT
REPORT
RIGHT
REPORT

expected result: output will show the pacman facing East, South, West and North again, sequentially.

---------------------

TEST 5: pacman rotates left
PLACE 1,1,NORTH
LEFT
REPORT
LEFT
REPORT
LEFT
REPORT
LEFT
REPORT

expected result: output will show the pacman facing West, South, East and North again, sequentially

---------------------

TEST 6: pacman is able to move in all four cardinal directions
PLACE 1,1,NORTH
MOVE
RIGHT
REPORT
MOVE
RIGHT
REPORT
MOVE
RIGHT
REPORT
MOVE
RIGHT
REPORT

expected result: output will be:

1,2,EAST
2,2,SOUTH
2,1,WEST
1,1,NORTH

showing that the pacman walked 1 unit north, 1 unit east, 1 unit south and 1 unit west, making a lap and ending up at the beginning

---------------------

TEST 7: pacman won't fall off the table
commands:
PLACE 4,4,NORTH
MOVE
MOVE
MOVE
MOVE
MOVE
REPORT

expected result: pacman will be at '4,4,NORTH', ignoring commands to move further

---------------------

TEST 8: pacman will ignore invalid commands
commands:
AUTODESTRUCT
TAKEOFF
KILL
REPORT

expected result: 'not in place', commands will be ignored, no errors will show
